                                                                            
uptime
    показывает время
uname -a
    показывает установленную систему (-a выводит в консоль все параметры)
lscpu
    показывает информацию о процессоре
ls -l
    выводит информацию о все директориях которые есть в данной директории
ls -a -l
    выводит информацию о все директориях которые есть в данной директории включая скрытые


echo "Hello World"
    выводит текст в консоль

echo $PATH
    выводит переменную окружения

echo $USER
    выводит имя пользователя в консоль

man -k "название команды"
    находит все команды по ключевому слову

man "название команды"
    находит конкретную команду и открывает ее в текстовом редакторе vi или nano

info "название команды"
    находит конкретную команду и открывает ее в текстовом редакторе info

whatis "название команды"
    находит конкретную команду и открывает ее выводит его описание в консоль

whereis "название команды"
    находит конкретную команду и открывает ее выводит путь к ней в консоль
uptime: /usr/bin/uptime (это место где размещается скрипт uptime)
        /usr/share/man/man1/uptime.1.gz (это место где размещается описание команды uptime)

locate "название директории или файла"
    поиск директорий и файлов в системе

plocate "название директории или файла"
    поиск директорий и файлов в системе (считается что использует более быстрый алгоритм)

ping <адрес>
    проверяет эхо-запросы на указанный IP-адрес (или доменное имя)

ping -c 3 <адрес>
    проверяет эхо-запросы на указанный IP-адрес (или доменное имя) 3 раза

ping -c 3 -i 1 <адрес>
ping -c 3 -i 1 google.com
    проверяет эхо-запросы на указанный IP-адрес (или доменное имя) 3 раза с интервалом 1 секунда

ps
    посмотреть какие процессоры запушенные у данного пользователя

ping google.com -> Ctrl Left + Z -> uptime or other commands
    -> fg - восстановить процесс -> Ctrl Left + C

fg
    восстановить процесс

ls -al -R /

or superuser:

sudo ls -al -R /
    выводит все файлы системы и всех подключенных дисков в консоль

Linux, в отличие от dos, НЕ ищет команду (или программу) в текущей директории,
только в $PATH! Чтобы запустить её, надо набрать ./название_программы
