### Создание алиаса для команды `robocopy` в PowerShell

1. **Открытие PowerShell**:
   Запустите PowerShell от имени администратора.

2. **Проверка профиля**:
   Убедитесь, что профиль существует:
   ```powershell
   echo $PROFILE
   ```
   Если файл не существует, создайте его:
   ```powershell
   New-Item -Path $PROFILE -ItemType File -Force
   ```

3. **Редактирование профиля**:
   Откройте профиль в Notepad:
   ```powershell
   notepad $PROFILE
   ```
   Добавьте следующую функцию для создания алиаса `acp`:
   ```powershell
   function acp { robocopy @args /E }
   ```
   Сохраните изменения и закройте редактор.

4. **Изменение политики выполнения**:
   Если вы получили ошибку о блокировке выполнения скриптов, измените политику выполнения:
   ```powershell
   Set-ExecutionPolicy RemoteSigned
   ```
   Подтвердите изменение, нажав `Y`.

5. **Перезагрузка профиля**:
   После внесения изменений выполните команду:
   ```powershell
   . $PROFILE
   ```

6. **Использование алиаса**:
   Теперь вы можете использовать команду `acp`:
   ```powershell
   acp "C:\source" "C:\destination"
   ```

### Примечания

- Если возникают проблемы с доступом к профилю, убедитесь, что вы редактируете правильный файл.
- Используйте команды `Get-Command acp` для проверки доступности функции.
- Для запуска скриптов без изменения политики выполнения, используйте:
  ```powershell
  powershell -ExecutionPolicy Bypass -File <path_to_script>
  ```
