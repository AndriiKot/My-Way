Для переименования ветки master в main с помощью командной строки (cmd) в Git выполните следующие шаги:

Проверьте текущую ветку: Убедитесь, что вы находитесь не на ветке, которую хотите переименовать (в данном случае, master). Если вы находитесь на ней, переключитесь на другую ветку, например, на develop:

 
git checkout develop
Переименуйте ветку: Выполните команду для переименования ветки:

 
git branch -m master main
Обновите ссылку на удалённый репозиторий: После локального переименования вам нужно обновить удалённый репозиторий. Сначала удалите старую ветку, а затем добавьте новую:

 
git push origin :master  # Удаляем старую ветку
git push origin main     # Пушим новую ветку
Установите новую ветку по умолчанию (если нужно): Если на удалённом репозитории нужно установить новую ветку main как ветку по умолчанию, это необходимо сделать через интерфейс управления репозиторием (например, на GitHub, GitLab и т.д.).

Обновите ссылку на ветку по умолчанию в локальном репозитории (если вы используете её в других репозиториях):

 
git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main
После выполнения этих шагов, ваш локальный и удалённый репозиторий будут использовать ветку main вместо master.
